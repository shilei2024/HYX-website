## 部署前准备

### 1. 文件完整性检查

确保以下核心文件存在：

```
HYX-Website/
├── index.html              # 首页
├── assets/                 # 静态资源
├── css/styles.css
├── js/
│   ├── components.js
│   └── data-loader.js
├── data/                   # 数据文件
├── products/               # 产品页面
├── about/                  # 关于我们
├── news/                   # 新闻中心
├── contact/                # 联系我们
├── en/                     # 英文版
├── ru/                     # 俄文版
├── Dockerfile              # Docker 配置
├── docker-compose.yml      # Docker Compose 配置
└── docker-deploy.sh        # 部署脚本
```

### 2. 配置信息确认

编辑 `data/config.json`，确认以下信息：

- [ ] 公司名称、简称
- [ ] 联系方式（电话、邮箱、联系人）
- [ ] 公司地址
- [ ] Logo 路径
- [ ] ICP 备案号（如有）
- [ ] 公安备案号（如有）
- [ ] Formspree 表单 ID（用于联系表单）

### 3. 本地测试

```bash
# 使用 Python
python -m http.server 8080

# 或使用 Docker 本地测试
docker build -t hyx-website .
docker run -d -p 8080:80 hyx-website
```

访问 http://localhost:8080 检查所有功能。

## Docker 部署（推荐）

### 方式一：一键部署脚本

```bash
# 1. 进入项目目录
cd HYX-website

# 2. 添加执行权限（Linux/Mac）
chmod +x docker-deploy.sh

# 3. 构建并启动
./docker-deploy.sh build
./docker-deploy.sh start

# 查看状态
./docker-deploy.sh status

# 查看日志
./docker-deploy.sh logs
```

### 方式二：Docker Compose

```bash
# 构建并启动
docker-compose up -d --build

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down

# 重启服务
docker-compose restart
```

### 方式三：手动 Docker 命令

```bash
# 1. 构建镜像
docker build -t hyx-website:latest .

# 2. 运行容器
docker run -d \
  --name hyx-website \
  --restart unless-stopped \
  -p 80:80 \
  -e TZ=Asia/Shanghai \
  hyx-website:latest

# 3. 查看状态
docker ps | grep hyx-website

# 4. 查看日志
docker logs -f hyx-website

# 5. 停止容器
docker stop hyx-website

# 6. 删除容器
docker rm hyx-website
```

### Docker 常用命令

| 命令 | 说明 |
|------|------|
| `docker ps` | 查看运行中的容器 |
| `docker logs -f hyx-website` | 查看实时日志 |
| `docker exec -it hyx-website sh` | 进入容器终端 |
| `docker restart hyx-website` | 重启容器 |
| `docker stop hyx-website` | 停止容器 |
| `docker rm hyx-website` | 删除容器 |
| `docker rmi hyx-website` | 删除镜像 |

## 其他部署方式

### 方式一：FTP/SFTP 上传

1. 使用 FTP 客户端连接服务器
2. 将所有文件上传到 web 根目录
3. 设置权限：文件 644，目录 755

### 方式二：Git 部署

```bash
cd /var/www
git clone <your-repo-url> hyx-website
cd hyx-website
chmod -R 755 .
chown -R www-data:www-data .
```

### 方式三：云服务托管

支持阿里云 OSS、腾讯云 COS、Vercel、Netlify 等静态托管服务。

## HTTPS配置

### 使用 Let's Encrypt

```bash
# 安装 Certbot
sudo apt install certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d your-domain.com -d www.your-domain.com

# 自动续期
sudo certbot renew --dry-run
```

## 常见问题解决

### 1. 容器启动失败

```bash
# 查看详细日志
docker logs hyx-website

# 检查端口占用
netstat -tlnp | grep 80
```

### 2. 页面 404 错误

- 检查文件是否正确复制到容器
- 查看 Nginx 配置是否正确

### 3. 数据加载失败

- 检查浏览器控制台错误
- 确认 `data/` 目录文件存在

### 4. 表单提交失败

- 在 [formspree.io](https://formspree.io) 配置表单 ID
- 更新 `data/config.json` 中的 `formspreeFormId`
