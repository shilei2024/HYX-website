# 部署说明文档

## 部署前准备

### 1. 检查文件完整性

确保以下文件和目录存在：

```
HYX-Website/
├── index.html              ✓
├── assets/                 ✓
├── css/                    ✓
├── js/                     ✓
├── data/                   ✓
├── products/               ✓
├── about/                  ✓
├── news/                   ✓
├── contact/                ✓
├── en/                     ✓
├── ru/                     ✓
└── README.md               ✓
```

### 2. 更新配置信息

编辑 `data/config.json`，确认以下信息正确：

- 公司名称
- 联系方式（电话、邮箱）
- 地址
- Logo 路径

### 3. 测试本地运行

```bash
# 使用 Python
python -m http.server 8080

# 或使用 Node.js
npx serve .
```

访问 http://localhost:8080 检查所有功能是否正常。

## 部署方式

### 方式 1：FTP/SFTP 上传（传统主机）

1. 使用 FTP 客户端（如 FileZilla）连接到服务器
2. 将所有文件上传到 web 根目录（通常是 `public_html` 或 `www`）
3. 确保文件权限正确（文件 644，目录 755）

**FileZilla 设置**：
- 主机：your-domain.com
- 用户名：ftp_username
- 密码：ftp_password
- 端口：21（FTP）或 22（SFTP）

---

### 方式 2：Git 部署（推荐）

如果服务器支持 Git：

```bash
# 在服务器上
cd /var/www
git clone <your-repo-url> hyx-website
cd hyx-website

# 设置权限
chmod -R 755 .
chown -R www-data:www-data .
```

**自动部署脚本**（可选）：

创建 `deploy.sh`：

```bash
#!/bin/bash
cd /var/www/hyx-website
git pull origin main
chmod -R 755 .
chown -R www-data:www-data .
echo "部署完成！"
```

---

### 方式 3：Docker 部署

```bash
# 构建镜像
docker build -t hyx-website .

# 运行容器
docker run -d -p 80:80 --name hyx-site hyx-website

# 查看日志
docker logs -f hyx-site
```

**Docker Compose**（可选）：

创建 `docker-compose.yml`：

```yaml
version: '3'
services:
  web:
    build: .
    ports:
      - "80:80"
    restart: always
    volumes:
      - ./logs:/var/log/nginx
```

运行：

```bash
docker-compose up -d
```

## Nginx 服务器配置

### 1. 安装 Nginx

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install nginx

# CentOS/RHEL
sudo yum install nginx
```

### 2. 配置网站

```bash
# 创建配置文件
sudo nano /etc/nginx/sites-available/hyx-website

# 粘贴 nginx.conf 中的内容
# 修改 server_name 和 root 路径
```

### 3. 启用网站

```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/hyx-website /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
```

### 4. 配置防火墙

```bash
# Ubuntu
sudo ufw allow 'Nginx HTTP'
sudo ufw allow 'Nginx HTTPS'

# CentOS
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload
```

## Apache 服务器配置

### 1. 安装 Apache

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install apache2

# CentOS/RHEL
sudo yum install httpd
```

### 2. 配置网站

```bash
# 复制网站文件
sudo cp -r . /var/www/html/

# 复制.htaccess
sudo cp .htaccess /var/www/html/

# 设置权限
sudo chown -R www-data:www-data /var/www/html
sudo chmod -R 755 /var/www/html
```

### 3. 启用模块

```bash
# 启用 rewrite 模块
sudo a2enmod rewrite
sudo a2enmod deflate
sudo a2enmod expires

# 重启 Apache
sudo systemctl restart apache2
```

## HTTPS/SSL 配置

### 使用 Let's Encrypt（免费）

```bash
# 安装 Certbot
sudo apt install certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d your-domain.com -d www.your-domain.com

# 自动续期
sudo certbot renew --dry-run
```

### 定时续期任务

```bash
# 编辑 crontab
sudo crontab -e

# 添加每 3 个月自动续期
0 0 1 1,4,7,10 * certbot renew
```

## 常见问题解决

### 1. 页面 404 错误

**原因**：服务器配置问题或文件路径错误

**解决**：
- 检查文件是否正确上传
- 检查服务器 rewrite 规则
- 查看 Nginx/Apache 错误日志

### 2. Logo 不显示

**原因**：路径错误或文件权限问题

**解决**：
- 检查 `data/config.json` 中的 logo 路径
- 确认 `assets/logo.png` 文件存在
- 检查文件权限（应为 644）

### 3. 数据加载失败

**原因**：JSON 文件路径错误或 CORS 问题

**解决**：
- 检查浏览器控制台错误
- 确认 `data/` 目录文件存在
- 检查服务器 MIME 类型配置

### 4. 移动端显示异常

**原因**：Bootstrap CDN 加载失败或 CSS 问题

**解决**：
- 检查网络连接
- 确认 Bootstrap CDN 可访问
- 清除浏览器缓存
