# 弘易芯科技官网 - 部署指南

## 📋 部署前准备

### 1. 服务器要求

- ✅ Linux 服务器（Ubuntu/CentOS/Debian）
- ✅ Docker 已安装
- ✅ Docker Compose v2.23.0+（推荐 v5.0+）
- ✅ Git 已安装
- ✅ 端口 80 可用（启用 HTTPS 时需 443 可用）

### 2. 检查环境

```bash
# 检查 Docker
docker --version

# 检查 Docker Compose
docker compose version

# 检查 Git
git --version
```

---

## 🚀 部署步骤

### 步骤 1：获取代码

**方式 A：使用 Git（推荐）**

```bash
# 创建网站目录
sudo mkdir -p /var/www/hyx-website
sudo chown $USER:$USER /var/www/hyx-website
cd /var/www/hyx-website

# 克隆代码
git clone <your-repo-url> .
```

**方式 B：上传文件**

```bash
# 在本地打包
tar -czf hyx-website.tar.gz ./HYX-website

# 上传到服务器
scp hyx-website.tar.gz user@server:/tmp/

# 在服务器上解压
ssh user@server
mkdir -p /var/www/hyx-website
tar -xzf /tmp/hyx-website.tar.gz -C /var/www/hyx-website --strip-components=1
cd /var/www/hyx-website
```

---

### 步骤 2：设置权限

```bash
# 为部署相关脚本添加执行权限（克隆或 git reset --hard 后若无可执行位需执行）
chmod +x 一键部署.sh manage-watch.sh install-service.sh setup-permissions.sh

# 可选：执行权限设置脚本，统一处理所有脚本权限
./setup-permissions.sh
```

> 若执行 `./一键部署.sh` 出现 **Permission denied**，可先运行 `chmod +x 一键部署.sh`，或使用 `bash 一键部署.sh`。

---

### 步骤 3：首次部署

```bash
# 执行一键部署脚本
./一键部署.sh
# 若提示 Permission denied，改用：bash 一键部署.sh
```

脚本会自动完成：
1. ✅ 检查 Docker 环境
2. ✅ 创建 `html/` 目录并复制网站文件
3. ✅ 构建 Docker 镜像
4. ✅ 启动容器
5. ✅ 询问是否启动 Watch 模式

---

### 步骤 4：启动 Watch 模式（推荐）

部署脚本会询问是否启动 Watch，输入 `y` 即可。

**或者手动启动：**

```bash
# 后台启动 Watch
./manage-watch.sh start-bg

# 查看状态
./manage-watch.sh status
```

**或安装系统服务（开机自启）：**

```bash
# 安装 systemd 服务
sudo ./install-service.sh

# 查看服务状态
sudo systemctl status hyx-watch
```

---

### 步骤 5：验证部署

```bash
# 检查容器状态
docker compose ps

# 查看容器日志
docker compose logs --tail=20 hyx-website

# 测试网站访问
curl http://localhost/

# 或在浏览器访问
http://your-server-ip
```

---

### 步骤 6：启用 HTTPS（443，可选）

若需 SSL 证书与 443 端口：

1. **开放 443 端口**：防火墙与云安全组放行 443/tcp。
2. **按 [HTTPS配置指南.md](HTTPS配置指南.md)** 申请证书，将 `fullchain.pem`、`privkey.pem` 放入项目 `ssl/` 目录，并修改 `nginx-https.conf` 中的 `server_name` 为你的域名。
3. **启动 HTTPS**（二选一）：
   - 再次执行 `./一键部署.sh`，在提示「是否启用 HTTPS（443 端口）」时选 `y`；
   - 或手动执行：`docker compose down` 后 `docker compose -f docker-compose-https.yml up -d --build`。

## ⚙️ 配置说明

### docker-compose.yml 关键配置

```yaml
services:
  hyx-website:
    volumes:
      - ./html:/usr/share/nginx/html:rw  # 同步目录
    
    develop:
      watch:
        # 静态文件同步
        - path: ./html/
          action: sync
          target: /usr/share/nginx/html
          ignore:
            - logs/
            - .git/
        
        # Nginx 配置同步并重启
        - path: ./nginx-docker.conf
          action: sync+restart
          target: /etc/nginx/conf.d/default.conf
```

### 同步规则

| 文件类型 | 同步方式 | 生效时间 |
|---------|---------|---------|
| HTML、CSS、JS | 实时同步 | 立即 |
| 图片文件 | 实时同步 | 立即 |
| Nginx 配置 | 同步 + 重启 | 5 秒内 |
| `.git/`、`logs/` | 忽略 | - |

## 🔧 常用命令

### 容器管理

```bash
# 查看状态
docker compose ps

# 查看日志
docker compose logs -f hyx-website

# 重启容器
docker compose restart hyx-website

# 停止容器
docker compose down
```

### Watch 管理

```bash
# 查看状态
./manage-watch.sh status

# 后台启动
./manage-watch.sh start-bg

# 停止
./manage-watch.sh stop

# 查看日志
./manage-watch.sh logs
```

### systemd 服务（如已安装）

```bash
# 查看状态
sudo systemctl status hyx-watch

# 重启服务
sudo systemctl restart hyx-watch

# 查看日志
sudo journalctl -u hyx-watch -f
```
