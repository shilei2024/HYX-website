# 弘易芯科技官网 - 故障排查指南

## 🔍 常见问题

### 问题 1：Watch 模式不工作

**症状**：`git pull` 后网站没有自动更新

**排查步骤**：

```bash
# 1. 检查 Watch 是否运行
./manage-watch.sh status

# 2. 检查 Docker Compose 版本
docker-compose version
# 需要 v2.23.0 或更高

# 3. 手动启动 Watch 查看错误
docker-compose watch

# 4. 查看 Watch 日志
tail -f watch.log
```

**解决方案**：

```bash
# 重启 Watch 模式
./manage-watch.sh restart

# 或重新安装服务
sudo ./install-service.sh
```

### 问题 2：文件同步但不生效

**症状**：文件已同步到容器，但浏览器看不到变化

**排查步骤**：

```bash
# 1. 检查容器内文件
docker exec hyx-website ls -lh /usr/share/nginx/html/

# 2. 检查文件内容
docker exec hyx-website cat /usr/share/nginx/html/index.html | head -20

# 3. 检查 Nginx 配置
docker exec hyx-website nginx -t

# 4. 查看 Nginx 日志
docker-compose logs --tail=50 hyx-website
```

**解决方案**：

```bash
# 1. 强制刷新浏览器缓存
# Windows: Ctrl + Shift + R
# Mac: Cmd + Shift + R

# 2. 重启容器
docker-compose restart hyx-website

# 3. 检查文件权限
chmod -R 755 html/
```

### 问题 3：容器无法启动

**症状**：`docker-compose up -d` 失败

**排查步骤**：

```bash
# 1. 查看详细错误
docker-compose up

# 2. 检查端口占用
sudo netstat -tlnp | grep :80

# 3. 检查 Docker 日志
sudo journalctl -u docker -n 50
```

**解决方案**：

```bash
# 1. 停止占用端口的服务
sudo systemctl stop nginx  # 如果安装了 nginx

# 2. 清理旧容器
docker-compose down --remove-orphans

# 3. 重新启动
docker-compose up -d
```

### 问题 4：Git pull 失败

**症状**：`git pull` 报错

**常见错误及解决**：

```bash
# 错误：本地修改冲突
# 解决：暂存本地修改
git stash
git pull
git stash pop

# 错误：权限问题
# 解决：修复目录权限
sudo chown -R $USER:$USER /var/www/hyx-website

# 错误：找不到分支
# 解决：检查分支名
git branch -a
git pull origin main  # 或 master
```

### 问题 5：网站访问 403/404

**症状**：浏览器显示 403 Forbidden 或 404 Not Found

**排查步骤**：

```bash
# 1. 检查文件是否存在
docker exec hyx-website ls -lh /usr/share/nginx/html/index.html

# 2. 检查 Nginx 配置
docker exec hyx-website cat /etc/nginx/conf.d/default.conf

# 3. 查看错误日志
docker-compose logs hyx-website | grep error
```

**解决方案**：

```bash
# 1. 重新复制文件到 html 目录
rm -rf html/
mkdir html
rsync -av --exclude='html/' --exclude='logs/' . html/

# 2. 重启容器
docker-compose restart hyx-website
```

### 问题 6：内存/CPU 占用过高

**症状**：服务器资源占用高

**排查步骤**：

```bash
# 查看容器资源使用
docker stats

# 查看系统资源
top
htop
```

**解决方案**：

```bash
# 1. 重启容器
docker-compose restart hyx-website

# 2. 限制容器资源（编辑 docker-compose.yml）
# 添加：
#   deploy:
#     resources:
#       limits:
#         cpus: '0.5'
#         memory: 256M

# 3. 清理未使用的资源
docker system prune -f
```

---

### 问题 7：日志文件过大

**症状**：磁盘空间不足

**排查步骤**：

```bash
# 查看磁盘使用
df -h

# 查看日志文件大小
ls -lh logs/
```

**解决方案**：

```bash
# 1. 清理日志文件
> logs/access.log
> logs/error.log

# 2. 配置日志轮转
# 编辑 docker-compose.yml，添加：
#   logging:
#     driver: "json-file"
#     options:
#       max-size: "10m"
#       max-file: "3"

# 3. 重启容器
docker-compose up -d
```

## 🔧 高级故障排查

### 查看容器详细信息

```bash
# 容器详细信息
docker inspect hyx-website

# 容器进程
docker top hyx-website

# 容器资源使用
docker stats hyx-website
```

### 进入容器调试

```bash
# 进入容器 shell
docker exec -it hyx-website /bin/sh

# 在容器内检查
ls -lh /usr/share/nginx/html/
cat /etc/nginx/conf.d/default.conf
nginx -t
```

### 网络问题排查

```bash
# 检查容器网络
docker network ls

# 检查端口映射
docker port hyx-website

# 测试容器内网络
docker exec hyx-website wget -q --spider http://localhost/
echo $?  # 应返回 0
```

## 🛠️ 重置部署（终极方案）

如果以上方法都无效，可以完全重置：

```bash
# 1. 停止并清理
docker-compose down --remove-orphans
docker-compose down --rmi all

# 2. 备份重要文件
cp -r html/ html.backup.$(date +%Y%m%d)
cp docker-compose.yml docker-compose.yml.bak

# 3. 重新部署
./一键部署.sh

# 4. 启动 Watch
./manage-watch.sh start-bg
```
