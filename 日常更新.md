# 弘易芯科技官网 - 日常更新指南

## 🎯 日常更新流程

### 已启动 Watch 模式（推荐）

```bash
# 1. 登录服务器
ssh user@your-server-ip

# 2. 进入网站目录
cd /var/www/hyx-website

# 3. 拉取最新代码
git pull

# 完成！网站会在 3-5 秒内自动更新
```

### 未启动 Watch 模式

```bash
# 1. 拉取代码
git pull

# 2. 重启容器
docker compose restart hyx-website
```

## 📦 完整更新示例

### 示例 1：更新首页内容

```bash
# 在本地修改文件
# 修改 index.html

# 提交并推送
git add index.html
git commit -m "更新首页内容"
git push

# 在服务器上
ssh user@server
cd /var/www/hyx-website
git pull

# 等待 3 秒，刷新浏览器查看效果
```

### 示例 2：更新 CSS 样式

```bash
# 在本地修改

git push

# 在服务器上
ssh user@server
cd /var/www/hyx-website
```

## 🔍 验证更新

### 检查文件是否同步

```bash
# 查看 html 目录文件时间戳
ls -lh html/

# 查看最新修改的文件
ls -lt html/ | head -10
```

### 检查容器日志

```bash
# 实时查看日志
docker compose logs -f hyx-website

# 查看最近 20 行
docker compose logs --tail=20 hyx-website
```

### 检查 Watch 状态

```bash
# 查看 Watch 状态
./manage-watch.sh status

# 查看 Watch 日志
tail -f watch.log
```

## 🛠️ 常用维护命令

### 查看容器状态

```bash
# 查看所有容器
docker compose ps

# 查看详细信息
docker inspect hyx-website
```

### 重启服务

```bash
# 重启容器
docker compose restart hyx-website

# 重启所有容器
docker compose restart
```

### 清理空间

```bash
# 清理未使用的镜像
docker image prune -f

# 清理未使用的容器
docker container prune -f

# 清理所有未使用的资源
docker system prune -f
```

### 查看资源使用

```bash
# 查看容器资源使用
docker stats hyx-website

# 查看磁盘使用
docker system df
```

## 📊 更新监控

### 设置简单监控

```bash
# 创建监控脚本
cat > monitor.sh << 'EOF'
#!/bin/bash
echo "=== 容器状态 ==="
docker compose ps

echo ""
echo "=== 最近日志 ==="
docker compose logs --tail=10 hyx-website

echo ""
echo "=== 磁盘使用 ==="
df -h /var/www
EOF

chmod +x monitor.sh

# 运行监控
./monitor.sh
```

### 定期检查项目

- ✅ 容器是否正常运行：`docker compose ps`
- ✅ 日志是否有错误：`docker compose logs --tail=50`
- ✅ 磁盘空间是否充足：`df -h`
- ✅ Watch 模式是否运行：`./manage-watch.sh status`

## ⚠️ 注意事项

### 1. 更新前备份

```bash
# 备份重要配置
cp docker-compose.yml docker-compose.yml.bak
cp -r html/ html.backup.$(date +%Y%m%d)
```

### 2. 避免高峰期更新

- 选择业务低峰期进行更新
- 重大更新前先测试

### 3. 浏览器缓存

更新后如果看不到效果：

```bash
# 强制刷新浏览器
# Windows: Ctrl + Shift + R
# Mac: Cmd + Shift + R

# 或清除浏览器缓存
```

### 4. 回滚操作

```bash
# 如果更新有问题，可以回滚到备份
git reset --hard <previous-commit>

# 或从备份恢复
cp -r html.backup.20250101/* html/

# 重启容器
docker compose restart hyx-website
```
